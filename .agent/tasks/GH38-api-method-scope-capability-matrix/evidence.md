## Acceptance Criteria Evidence

### AC1. Auth-model alignment

Status: PASS

Proof:
- `references/yandex-office-auth-principles.md` is the permanent auth-model
  source of truth.
- `spec.md` embeds the PR32 entity model and adds `api_method` as the missing
  entity.
- `capabilities/README.md` restates the user/agent/yandex-office/app/scope/token
  relationship and keeps the matrix as capability/remediation data.

### AC2. Full method inventory

Status: PASS

Proof:
- `capabilities/methods.json` contains 315 method rows across Calendar,
  Contacts, Directory, Disk, Forms, Mail, Telemost, and Tracker.
- Cloud was removed from the yandex-office skill surface and extracted into the
  private standalone `yandex-cloud` repo.
- Every method row has the required classification vocabulary. Current counts:
  48 `current_used`, 11 `implemented_unused`, 256 `upstream_available`.

### AC3. Scope list for token minting

Status: PASS

Proof:
- `references/yandex-oauth-scopes.json` is the canonical OAuth scope list.
- `capabilities/atomic.client_id.json` contains tested atomic-token client ids.
- `capabilities/atomic.token` is the local ignored scope-to-token file.
- Validation reports 134 canonical OAuth scopes and 32 local auth contexts.

### AC4. Live probe process

Status: PASS

Proof:
- `capabilities/probe.py` is the single generic probe wrapper.
- Service-specific request shapes live in `capabilities/methods.json`.
- `capabilities/README.md` documents the mandatory process for maintaining and
  rerunning the map.

### AC5. Empirical capability matrix

Status: PASS with documented residuals

Proof:
- `capabilities/matrix.json` is generated by `capabilities/probe.py`.
- Every one of the 315 inventoried methods has at least one matrix cell.
- Cells use the allowed status vocabulary.
- `capabilities/method-scope-map.json` is generated from successful cells and
  contains 309 methods with `public`, `one_of`, or `all_of` capability entries.

Residuals:
- Five SMTP methods have only `unclear_needs_retest` cells because this host
  could not reach Yandex SMTP.
- `mail.imap.uid.delete_matching.live_realty` is an explicit negative/safety
  probe for `mail:imap_ro`; it has no successful scope result and is therefore
  absent from `method-scope-map.json`.

### AC6. Runtime consumption guidance

Status: PASS for guidance, NOT IMPLEMENTED for runtime token selection

Proof:
- `spec.md` defines runtime consumption as a capability/remediation layer.
- `capabilities/README.md` preserves the PR32 rule that live API responses remain
  final truth.
- Mail IMAP read/write caveats and runtime guardrail implications are documented
  separately in `capabilities/mail-imap-scenarios.md`.
- `notes/runtime-method-scope-map-integration.md` records the current runtime gap
  and the required changes to use `method-scope-map.json` for deterministic
  token choice.

Gap:
- current callers still pass broad hardcoded `required_scopes` values into
  `resolve_token()`
- current token storage exposes one canonical `token.<service>` value, so it
  cannot choose among multiple tokens for the same service

### AC7. Secret handling

Status: PASS

Proof:
- `capabilities/validate.py` checks capability artifacts and task artifacts for
  raw Yandex OAuth token-shaped strings.
- `capabilities/validate.py` also checks `capabilities/raw/*.json` for raw OAuth
  tokens, unredacted Yandex diagnostic ids, and obvious live object ids.
- Live fixture emails/domains are represented as placeholders in
  `methods.json`; real values belong only in ignored `capabilities/probe.local.json`.
- `raw/validate.txt` shows validation passed on 2026-04-20.
